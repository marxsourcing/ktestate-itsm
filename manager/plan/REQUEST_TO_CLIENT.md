## 📋 요청 항목

### 1. IT 시스템 목록 (20개) — 🔴 긴급

| 항목 | 설명 | 예시 |
|------|------|------|
| 시스템 목록 | 20개 시스템의 **공식 명칭** | ONE, SAP ERP, 경영정보(BIS) 등 |
| 모듈 목록 | 시스템별 하위 모듈/기능 | ONE → 인사관리, 급여관리, 근태관리 등 |
| 시스템 분류 | 시스템 그룹핑 기준 | 핵심/업무/그룹웨어/홈페이지/플랫폼 등 |
| 담당자 정보 | 시스템별 기본 담당자 | (자동 배정 규칙용, 선택사항) |

```
예시:

|---------|------|-------|-------|-------|--------|
| ONE | 핵심 | 인사관리 | 급여관리 | 근태관리 | 홍길동 |
| SAP ERP | 핵심 | 재무 | 구매 | 생산 | 김철수 |
```

### 2. 요구사항 분류 체계 — 🔴 긴급

요구사항 접수 양식 및 DB 설계에 필요합니다.

| 항목 | 확정 필요 사항 | 현재 예상값 |
|------|--------------|------------|
| **요구사항 유형** | 공식 분류 확정 | 기능추가 / 기능개선 / 버그수정 / 기타 |
| **우선순위** | 등급 체계 확정 | 긴급 / 높음 / 보통 / 낮음 |
| **처리 상태** | 상태값 확정 | 접수 / 검토중 / 진행중 / 완료 / 반려 / 보류 |

**질문**: 위 예상값을 그대로 사용해도 괜찮으신지, 또는 기존에 사용하시는 분류 체계가 있으시면 공유 부탁드립니다.

---

### 3. 기존 요구사항 이력 데이터 — 🟡 중요

AI 핵심 기능인 **"유사 요구사항 검색"**, **"중복 요청 탐지"**, **"처리 답변 초안 생성"** 개발에 필요합니다.

| 항목 | 설명 |
|------|------|
| **용도** | RAG(검색증강생성) 학습 데이터로 활용 |
| **필요 데이터** | 과거 IT 요구사항 접수/처리 이력 |
| **최소 수량** | 100건 이상 권장 (많을수록 AI 품질 향상) |
| **형식** | Excel / CSV / 기존 시스템 Export |

**필요한 컬럼 (가능한 범위 내)**:
- 요청 제목
- 요청 내용 (상세)
- 대상 시스템/모듈
- 요구사항 유형
- 처리 결과/답변
- 처리 일자

**참고**: 개인정보가 포함된 경우 마스킹 처리 후 전달 부탁드립니다.
