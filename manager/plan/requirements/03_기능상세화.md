## 전체 UI 컨셉: ChatGPT 스타일

모든 주요 화면은 **좌측 목록 패널 + 우측 AI 채팅 영역** 레이아웃을 따릅니다.

```
┌─────────────────────────────────────────────────────────────┐
│ 헤더                                                         │
├────────────────┬────────────────────────────────────────────┤
│  사이드 패널    │        메인 콘텐츠 (AI 채팅)                 │
│  (대화/요청    │        ┌────────────────────────────────┐  │
│   목록)        │        │ 대화 히스토리 (스크롤)          │  │
│                │        │                                │  │
│  + 새 대화     │        │ AI: 무엇을 도와드릴까요?        │  │
│  ─────────    │        │ 사용자: 급여조회 개선...        │  │
│  대화 1       │        │ AI: 분석 결과입니다            │  │
│  대화 2       │        │ [요구사항 카드 미리보기]        │  │
│  ...          │        └────────────────────────────────┘  │
│                │        [입력창...               ] [전송]   │
├────────────────┴────────────────────────────────────────────┤
│ 액션바 (요구사항 확정 / 상태 변경 / 담당자 배정)                │
└─────────────────────────────────────────────────────────────┘
```

## AI 채팅 시스템 상세 (요청자용)

### 메인 채팅 인터페이스 (`/chat`)

**레이아웃 구성**:
- **좌측 사이드바** (280px)
  - 새 대화 버튼
  - 대화 목록 (최신순, 무한 스크롤)
  - 각 대화 항목: 제목, 상태 뱃지 (진행중/확정됨), 날짜
  - 대화 삭제/이름 변경 옵션
  
- **우측 채팅 영역**
  - 대화 히스토리 (메시지 버블)
  - AI 메시지: 텍스트 + 요구사항 카드 + 추가 질문
  - 사용자 메시지: 텍스트 + 첨부파일
  - 입력 영역: 텍스트박스 + 파일 첨부 + 전송 버튼

- **하단 액션바** (대화가 충분히 진행된 경우)
  - "요구사항으로 확정" 버튼
  - 요구사항 카드 수정 옵션

**채팅 흐름**:
1. 사용자가 자연어로 요구사항 설명
2. AI가 실시간으로 분석하며 응답
3. 부족한 정보가 있으면 AI가 추가 질문
4. 충분한 정보가 모이면 AI가 요구사항 카드 생성
5. 카드에서 수정 가능 (시스템/모듈/유형/제목/내용)
6. "확정" 시 요구사항 보드로 이동

### AI 추가 질문 생성

**동작 방식**:
- 요구사항 완성도 분석 (시스템, 모듈, 상세 내용 등)
- 부족한 항목에 대해 자연스러운 질문 생성
- 예시:
  - "어떤 화면에서 이 기능을 사용하시나요?"
  - "현재는 어떻게 처리하고 계신가요?"
  - "급한 정도는 어느 정도인가요?"

**UI**:
- AI 메시지 버블에 질문 표시
- 사용자 답변 시 요구사항 카드 자동 업데이트

## 유사 요구사항 검색 시스템 (RAG) 상세

### 유사 요구사항 검색

**화면/작업 구성**:
- 유사 요구사항 목록: 검색된 유사 건 카드 목록 (제목, 상태, 일자)
- 유사도 표시: 각 건별 유사도 퍼센트
- 유사 건 상세 모달: 상세 내용, 처리 답변 확인

**주요 처리**:
- 입력된 요구사항 벡터 임베딩 생성
- 벡터 DB에서 유사 요구사항 검색
- 유사도 계산 및 정렬
- 검색 결과 표시

---

### 중복 탐지 및 알림

**화면/작업 구성**:
- 중복 경고 알림: "유사한 요청이 이미 있습니다" 메시지
- 기존 요청 링크: 해당 요청으로 이동 버튼
- 계속 진행 옵션: 그래도 접수하기 버튼

**주요 처리**:
- 유사도 임계값 이상 건 탐지
- 중복 가능성 판단
- 경고 메시지 표시
- 사용자 선택에 따른 처리 분기

### 요청 상세 화면 (`/requests/[id]`) - 요청자용

**레이아웃**:
```
┌──────────────┬─────────────────────────────────────────┐
│              │                                         │
│  요청 정보   │   📋 AI 분석 결과                        │

│  제목: ...   │   시스템: 인사/급여 시스템               │
│  시스템: ... │   모듈: 급여명세서 조회                  │
│  상태: 검토중│   유형: 기능개선                        │
│              │   ───────────────────────────────────   │
│  첨부파일    │   급여명세서 조회 기간을 전년도까지     │
│  - file1.png │   확장해주세요.                         │
│              │                                         │
│  처리 이력   ├─────────────────────────────────────────┤
│  ──────────  │   💬 댓글/메모                           │
│  - 12/25 접수│   ───────────────────────────────────   │
│  - 12/25 배정│   담당자: 화면 캡처 부탁드립니다        │
│              │   나: [screenshot.png 첨부]             │
│  처리 답변   │   ───────────────────────────────────   │
│  ──────────  │   [댓글 입력...                ] [전송]  │
│  "반영 완료" │                                         │
└──────────────┴─────────────────────────────────────────┘
```

**주요 기능**:
- 좌측: 요구사항 정보, 첨부파일, 처리 이력 타임라인, 처리 답변
- 우측 상단: AI 분석 결과 카드 (요구사항 확정 시 저장된 분석 결과만 표시)
- 우측 하단: 댓글/메모 영역 (담당자와 추가 정보 소통용)
- 상태 업데이트 실시간 반영

**⚠️ 채팅 가시성 제한**:
- 요청자는 자신이 AI와 대화한 원본 채팅 내역을 볼 수 없음 (분석 결과만 표시)
- 담당자의 내부 AI 협업 채팅도 볼 수 없음
- 담당자와의 소통은 "댓글/메모" 방식으로만 가능

## 처리담당자 시스템 상세

### 담당자 워크스페이스 (`/workspace`)

**⚠️ 이 화면의 AI 협업 채팅은 내부 전용이며, 요청자에게 공개되지 않습니다.**

**레이아웃**:
```
┌──────────────┬─────────────────────────────────────────┐
│              │   🔒 내부 AI 협업 채팅 (요청자 비공개)   │
```
