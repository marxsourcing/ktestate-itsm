# POC4: AI 기반 IT 요구사항관리 시스템 - 주요 개발 범위

## 🗂️ 주요 개발 범위

### 1. AI 챗봇 시스템

- 자연어 기반 요구사항 접수 인터페이스
- LLM 기반 시스템/모듈 자동 분류
- 요구사항 유형 자동 판단 (기능추가/개선/버그수정/기타)
- 불완전한 요구사항에 대한 추가 질문 생성
- 분류 결과 확인 및 수정 인터페이스

### 2. 유사 요구사항 검색 시스템 (RAG)

- 요구사항 벡터 임베딩 생성
- 유사 요구사항 자동 검색
- 중복 요청 탐지 및 알림
- 유사 사례 해결 방법 참조 제공

### 3. 처리 지원 AI 시스템

- 처리 답변 초안 자동 생성
- 기술 용어 → 사용자 친화적 표현 변환
- 유사 사례 기반 답변 추천

### 4. 사용자 시스템 (요청자)

- 임직원 로그인 (SSO 연동 고려)
- AI 챗봇 기반 요구사항 접수
- 첨부파일 업로드
- 내 요청 목록 조회
- 요청 상세 조회 (상태, 답변 확인)
- 추가 정보 제공 기능

### 5. 처리담당자 시스템

- 담당자 로그인
- 배정된 요청 목록 조회 (필터/검색)
- 요청 상세 조회
- AI 제공 유사 사례 참고
- 처리 상태 변경 (접수/진행중/완료/반려)
- 답변 작성 (AI 초안 지원)
- 전자결재 연동 (증적 관리)

### 6. 관리자 시스템

- 관리자 로그인
- 전체 요청 목록 조회 (필터/검색)
- 담당자 배정 및 변경
- 우선순위 조정
- 요청 현황 대시보드 (통계)
- 시스템/모듈 설정 관리 (20개 시스템)
- 담당자 배정 규칙 설정
- **전체 채팅 내역 조회 (관리자 전용)**
  - 요청자-AI 채팅 원본 조회
  - 담당자-AI 협업 채팅 조회
  - 감사/증적용 Excel 내보내기

### 7. 데이터 연계

- IT 시스템 목록 데이터 구축 (20개 시스템)
- 시스템별 모듈 데이터 구축
- 기존 요구사항 이력 데이터 마이그레이션 (선택)
- 전자결재 시스템 API 연동

### 8. 시스템 통합 및 배포

- 클라우드 인프라 구축 (개발/테스트 환경)
- 내부망 AI 모델 구축 (GPT OSS) 또는 외부 API 연동
- 벡터 데이터베이스 구축 (RAG용)
- CI/CD 파이프라인 구성
- 모니터링 및 로깅
- 사용자 가이드 작성
