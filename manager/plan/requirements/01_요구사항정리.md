# POC4: AI 기반 IT 요구사항관리 시스템 - 요구사항 정리

## 핵심 요구사항 정리

### 프로젝트 개요
- **프로젝트명**: AI 기반 IT 요구사항관리 시스템 POC
- **프로젝트 유형**: 웹 애플리케이션 (AI 기반)
- **목적**: IT 시스템 변경/개선 요청을 효율적으로 접수하고 관리하며, AI를 활용한 자동 분류 및 처리 지원
- **주요 사용자**: 
  - 요청자 (전 임직원)
  - 처리담당자 (IT팀)
  - 관리자 (IT팀장)

---

## 필수 구현 요구사항 (클라이언트 요청)

### 1. IT 시스템 선택 기능
- 총 20개 IT 시스템 선택 가능
- 시스템별 세부 모듈 선택 옵션

### 2. 요구사항 분류 체계
- 요구사항 구분 (기능추가/기능개선/버그수정/기타)
- 상세 분류 체계 (시스템별 카테고리)

### 3. 요청 관리
- 요청자/처리담당자/처리상태 관리
- 요청 제목 및 상세 내용 작성
- 첨부파일 기능

### 4. 요청 목록 조회 및 관리
- 요청 목록 조회
- 필터링/검색 기능
- 상태별 관리

### 5. 전자결재 연동
- 전자결재 시스템 연동을 통한 증적 관리

### 6. UX 요구사항
- 하나의 화면에서 요구사항 접수 가능

## AI 차별화 기능 (제안)

### 🔥 핵심 AI 기능

#### 1. 자연어 기반 요구사항 접수 (챗봇)
- 대화형 인터페이스로 요구사항 접수
- 복잡한 폼 입력 대신 자연어로 설명
- "하나의 화면에서 요구사항 접수" 요구사항을 AI 챗봇으로 구현
- 예시:
  ```
  사용자: "급여명세서 조회할 때 작년 데이터도 볼 수 있게 해주세요"
  
  → AI 자동 분석:
    - 시스템: 인사/급여 시스템
    - 모듈: 급여명세서 조회
    - 요구사항 유형: 기능 개선
    - 상세 내용: 급여명세서 조회 기간을 전년도까지 확장
  ```

#### 2. 시스템/모듈 자동 분류 & 라우팅
- 20개 시스템 중 적합한 시스템 자동 판단
- 세부 모듈 자동 추천
- 적합한 처리담당자 자동 배정 제안

#### 3. 유사 요구사항 검색 & 중복 탐지
- 과거 유사 요청 자동 검색 (RAG)
- "이미 처리된 유사 건이 있습니다" 알림
- 해결 방법/답변 참고 제공
- 중복 요청 방지

#### 4. 처리 답변 초안 생성
- 처리 완료 시 답변 초안 자동 생성
- 기술적 내용을 사용자 친화적 언어로 변환
- FAQ 기반 자동 응답 제안

### 💡 부가 AI 기능

#### 5. 요구사항 품질 체크 & 보완 제안
- 불완전한 요구사항에 대해 추가 정보 요청
- 예시: "어떤 화면에서 문제가 발생하나요?", "재현 조건이 있나요?"

#### 6. 영향도 분석 자동화 (선택)
- 변경이 다른 시스템에 미치는 영향 분석
- 관련 시스템/모듈 자동 탐지

## 시스템 구성

### 사용자 화면 (요청자)
1. **AI 챗봇 인터페이스** (메인)
   - 자연어로 요구사항 입력
   - AI가 분류 결과 확인 요청
   - 첨부파일 추가
   - 최종 제출

2. **내 요청 목록**
   - 본인이 요청한 목록 조회
   - 상태 확인 (접수/진행중/완료/반려)
   - 요청 상세 조회
   - 추가 정보 제공

### 처리담당자 화면 (IT팀)
1. **요청 목록 관리**
   - 배정된 요청 목록
   - 필터/검색
   - 우선순위 표시

2. **요청 처리**
   - 요청 상세 조회
   - 유사 사례 참고 (AI 제공)
   - 처리 상태 변경
   - 답변 작성 (AI 초안 지원)
   - 전자결재 연동

### 관리자 화면 (IT팀장)
1. **대시보드**
   - 요청 현황 통계
   - 담당자별 처리 현황
   - 시스템별 요청 분포

2. **요청 관리**
   - 전체 요청 목록
   - 담당자 배정/변경
   - 우선순위 조정

3. **시스템 설정**
   - IT 시스템 목록 관리 (20개)
   - 모듈 관리
   - 담당자 배정 규칙

## 기술 스택 (예상)

### AI/LLM
- **모델**: GPT OSS (내부망) / GPT-5 (외부 API)
- **RAG**: 유사 요구사항 검색용 벡터 DB (Chroma/Pinecone)
- **프레임워크**: LangChain

### Backend
- **프레임워크**: FastAPI (Python)
- **데이터베이스**: PostgreSQL

### Frontend
- **프레임워크**: Next.js + React
- **UI**: Tailwind CSS, shadcn/ui

### 연동
- **전자결재**: 기존 전자결재 시스템 API 연동

## POC 범위 vs 확장 범위

### POC 범위 (핵심)
- ✅ AI 챗봇 기반 요구사항 접수
- ✅ 시스템/모듈 자동 분류
- ✅ 유사 요구사항 검색
- ✅ 요청 목록 조회/관리
- ✅ 처리 상태 관리
- ✅ 처리 답변 초안 생성

### POC 범위 (선택)
- ⚠️ 전자결재 연동 (API 제공 여부에 따라)
- ⚠️ 영향도 분석 자동화

### 확장 범위 (POC 이후)
- 📋 리포트 자동 생성
- 📋 전사 시스템 확대 (20개 → 전체)
- 📋 AI 성능 지속 개선 (피드백 학습)

## 기대 효과

1. **사용자 경험 개선**
   - 복잡한 폼 대신 자연어로 간편 접수
   - AI가 자동 분류하여 입력 오류 감소

2. **처리 효율 향상**
   - 유사 사례 참고로 빠른 처리
   - 답변 초안 자동 생성으로 시간 절약
   - 자동 라우팅으로 배정 시간 단축

3. **데이터 품질 향상**
   - 일관된 분류 체계
   - 완전한 요구사항 작성 유도
   - 중복 요청 방지

4. **AI 기술 검증**
   - 내부 업무에 LLM 적용 가능성 검증
   - 기존 시스템 연계 영향도 낮음 (독립 구성)
   - 성공 시 다른 업무로 확장 가능
