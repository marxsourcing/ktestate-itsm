# CLIENT GUIDE: 외부 연동 요청 사항

## 요청 현황 요약

| 우선순위 | 항목 | 상태 | 기한 |
|---------|------|------|------|
| 🔴 긴급 | Supabase 프로젝트 정보 | ⬜ 대기 | Week 1 |
| 🔴 긴급 | 사용자 인증 방식 결정 | ⬜ 대기 | Week 1 |

## 🔴 긴급 요청 (Week 1 내 필요)

> 개발 시작 전 반드시 확보해야 하는 항목

### 1. Supabase 프로젝트 정보

| 항목 | 상태 | 값 |
|------|------|-----|
| 프로젝트 URL | ⬜ | |
| anon key | ⬜ | |
| service_role key | ⬜ | |
| 리전 | ⬜ | (권장: ap-northeast-1) |

**참고**: Supabase 프로젝트는 클라이언트 계정에서 생성하거나, 개발용으로 별도 생성 가능

### 2. 사용자 인증 방식

| 항목 | 상태 | 값 |
|------|------|-----|
| SSO 연동 여부 | ⬜ | Yes / No |
| SSO 방식 | ⬜ | SAML / OIDC / 기타 |
| IdP 정보 | ⬜ | (SSO 사용 시) |
| 테스트 계정 | ⬜ | 최소 3개 (요청자/담당자/관리자) |

**대안**: SSO 미확정 시 이메일/비밀번호 인증으로 개발 시작 가능

### 3. 개발/테스트 환경

| 항목 | 상태 | 값 |
|------|------|-----|
| 개발용 도메인 | ⬜ | (Vercel 기본 도메인 사용 가능) |
| 접근 IP 제한 여부 | ⬜ | Yes / No |
| VPN 필요 여부 | ⬜ | Yes / No |

## 🟡 중간 요청 (Week 3까지)

> 핵심 기능 개발에 필요한 항목

### 4. IT 시스템 목록 (20개)

| 항목 | 상태 | 비고 |
|------|------|------|
| 시스템 목록 (20개) | ⬜ | 공식 명칭 필요 |
| 시스템별 모듈 목록 | ⬜ | 계층 구조 |
| 시스템별 담당자 | ⬜ | 기본 배정 규칙용 |
| 시스템 분류 체계 | ⬜ | 그룹핑 기준 |

**현재 예상 목록** (requirements/01_요구사항정리.md 참조):
```
1. ONE (통합업무시스템)
2. SAP ERP
3. 경영정보(BIS)
4. 호텔멤버십(HMS)
5. 인재원(LIME)
... (총 20개)
```

### 5. 요구사항 분류 체계

| 항목 | 상태 | 값 |
|------|------|-----|
| 요구사항 유형 | ⬜ | 기능추가/기능개선/버그수정/기타 (확정 필요) |
| 우선순위 체계 | ⬜ | 긴급/높음/보통/낮음 (확정 필요) |
| 상태 체계 | ⬜ | 접수/진행중/완료/반려 등 |

### 6. 전자결재 시스템 연동 (선택)

| 항목 | 상태 | 비고 |
|------|------|------|
| API 스펙 문서 | ⬜ | REST API 기준 |
| 테스트 환경 URL | ⬜ | |
| 테스트 계정 | ⬜ | |
| 인증 방식 | ⬜ | API Key / OAuth / 기타 |

**POC 범위 확인 필요**: 전자결재 연동이 POC 필수인지 선택인지

## 요청 템플릿

### 클라이언트에게 보낼 요청 메일 초안

```
제목: [요구사항관리시스템 POC] 개발 착수를 위한 정보 요청

안녕하세요,

AI 기반 IT 요구사항관리 시스템 POC 개발을 위해 
아래 정보를 요청드립니다.

[긴급 - 1주차 내 필요]
1. Supabase 프로젝트 생성 또는 접근 정보
2. 사용자 인증 방식 결정 (SSO 여부)
3. 테스트 계정 발급 (요청자/담당자/관리자 역할별)

[중요 - 3주차까지]
4. IT 시스템 20개 공식 목록 및 모듈 체계
5. 전자결재 연동 API 스펙 (POC 범위 포함 시)

회신 부탁드립니다.
감사합니다.
```
